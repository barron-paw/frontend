# 安卓手机错误-101修复指南

## 错误说明

错误-101通常表示：
- **DNS解析失败**：无法解析域名
- **SSL/TLS握手失败**：无法建立安全连接
- **网络连接被阻止**：运营商或防火墙阻止连接
- **Cloudflare代理问题**：Cloudflare代理配置不正确

## 快速解决方案

### 方案1：手动设置DNS（最有效）

在安卓手机上：

1. **打开设置**
   - 进入"设置" → "网络和互联网" → "高级" → "私有DNS"

2. **设置私有DNS**
   - 选择"私有DNS提供商主机名"
   - 输入以下任一DNS服务器：
     - `dns.google`（Google DNS，推荐）
     - `1dot1dot1dot1.cloudflare-dns.com`（Cloudflare DNS）
     - `dns.quad9.net`（Quad9 DNS）

3. **保存并测试**
   - 点击"保存"
   - 重新连接网络（关闭并重新打开WiFi或移动数据）
   - 重新访问网站

### 方案2：检查Cloudflare SSL/TLS设置

如果使用Cloudflare代理，需要检查SSL/TLS设置：

1. **登录Cloudflare Dashboard**
   - 访问：https://dash.cloudflare.com/
   - 选择域名 `hypebot.top`

2. **检查SSL/TLS设置**
   - 进入 **SSL/TLS** → **概述**
   - 确保加密模式设置为：
     - **完全（严格）**（推荐）
     - 或 **完全**
   - **不要使用** "灵活"模式

3. **检查其他设置**
   - **始终使用HTTPS**：开启
   - **自动HTTPS重写**：开启
   - **最低TLS版本**：1.2或更高

### 方案3：检查Cloudflare代理状态

如果使用Cloudflare代理，可能需要调整：

1. **检查DNS记录**
   - 进入 **DNS** → **Records**
   - 找到 `hypebot.top` 和 `www.hypebot.top` 的记录
   - 检查代理状态：
     - **🟠 Proxied（橙色云朵）**：通过Cloudflare代理
     - **⚪ DNS only（灰色云朵）**：直接指向Vercel

2. **如果使用代理，确保配置正确**
   - SSL/TLS模式必须为"完全（严格）"
   - Vercel的SSL证书必须有效

3. **如果问题持续，尝试DNS only模式**
   - 将代理状态改为 **⚪ DNS only**
   - 等待5-10分钟让DNS传播
   - 重新测试

### 方案4：清除浏览器缓存和数据

1. **清除Chrome缓存**
   - 打开Chrome浏览器
   - 点击右上角三个点 → "设置"
   - 进入"隐私和安全" → "清除浏览数据"
   - 选择"高级"标签
   - 时间范围选择"全部时间"
   - 勾选"缓存的图片和文件"和"Cookie及其他网站数据"
   - 点击"清除数据"

2. **清除DNS缓存**
   - 在安卓手机上，重启设备（最简单的方法）
   - 或者使用第三方应用清除DNS缓存

### 方案5：使用不同的网络

尝试：
- **切换到WiFi**（如果之前使用移动数据）
- **切换到移动数据**（如果之前使用WiFi）
- **使用不同的WiFi网络**
- **使用VPN**（已确认有效）

## 详细诊断步骤

### 步骤1：检查DNS解析

在安卓手机上，使用浏览器访问：
- `https://www.hypebot.top`
- 查看具体错误信息

如果显示"DNS解析失败"或"无法解析域名"：
- 使用方案1（手动设置DNS）

### 步骤2：检查SSL证书

如果显示"证书错误"或"连接不安全"：
- 使用方案2（检查Cloudflare SSL/TLS设置）

### 步骤3：检查网络连接

如果显示"网络错误"或"连接超时"：
- 使用方案5（使用不同的网络）
- 检查是否有防火墙或代理阻止连接

### 步骤4：检查Cloudflare配置

如果以上都不行：
- 使用方案3（检查Cloudflare代理状态）
- 考虑暂时使用DNS only模式

## 常见原因和解决方案

### 原因1：DNS解析失败

**症状**：无法解析域名 `hypebot.top`

**解决方案**：
- 手动设置DNS为 `dns.google` 或 `1.1.1.1`
- 或使用VPN（已确认有效）

### 原因2：SSL/TLS握手失败

**症状**：可以解析域名，但无法建立安全连接

**解决方案**：
- 检查Cloudflare的SSL/TLS设置
- 确保使用"完全（严格）"模式
- 检查Vercel的SSL证书是否有效

### 原因3：运营商限制

**症状**：某些运营商可能限制访问某些域名

**解决方案**：
- 使用VPN（已确认有效）
- 切换到不同的网络（WiFi vs 移动数据）
- 联系运营商解除限制

### 原因4：Cloudflare代理问题

**症状**：Cloudflare代理配置不正确

**解决方案**：
- 检查Cloudflare的SSL/TLS设置
- 确保Vercel的SSL证书有效
- 考虑暂时使用DNS only模式

## 推荐操作顺序

1. **首先尝试**：手动设置DNS（方案1）- 这是最常见的解决方案
2. **如果不行**：检查Cloudflare SSL/TLS设置（方案2）
3. **如果还不行**：尝试使用不同的网络（方案5）
4. **最后手段**：使用VPN（已确认有效）

### Cloudflare配置

- [ ] SSL/TLS模式设置为"完全（严格）"
- [ ] "始终使用HTTPS"已开启
- [ ] DNS记录正确指向Vercel
- [ ] 代理状态正确（Proxied或DNS only）

### Vercel配置

- [ ] 域名已正确添加
- [ ] SSL证书已正确配置

## 如果问题仍然存在

如果以上方案都不行，请提供以下信息：

1. **错误详情**：
   - 完整的错误信息
   - 错误代码（如果有）

2. **网络信息**：
   - 使用的网络类型（WiFi/移动数据）
   - 运营商名称（如果是移动数据）

3. **设备信息**：
   - 安卓版本
   - 浏览器名称和版本

4. **测试结果**：
   - 是否尝试了手动设置DNS
   - 是否尝试了不同的网络
   - 是否尝试了不同的浏览器
